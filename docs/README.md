# OctoAcme Project Management Docs

This README provides links to all project management process documents and a brief summary of the project management processes used by OctoAcme.

---

## Project Management Processes Summary

OctoAcme follows a structured, iterative project management approach built around five core phases:

1. **Initiation** — Validate the business need, align stakeholders, define success criteria, and produce a lightweight Project One-pager before committing to full planning.
2. **Planning** — Break approved work into a prioritized backlog with acceptance criteria, estimate scope, define the Definition of Done, and create a release plan and milestone map.
3. **Execution & Tracking** — Deliver in short iterations using daily standups, sprint reviews, a project board (Backlog → Done), and PR-based code reviews with CI automation. Risks and blockers are escalated through a three-level path from team triage to sponsor.
4. **Release & Deployment** — Release through a standardized checklist covering staging smoke tests, production deployment, rollback planning, and stakeholder announcements. Releases are categorized as Patch, Minor, or Major.
5. **Retrospective & Continuous Improvement** — After each sprint or milestone capture what went well, what could improve, and 2–3 actionable items to feed back into the backlog.

Cross-cutting concerns — **Risk Management** (a running Risk Register reviewed weekly) and **Stakeholder Communication** (weekly status updates, milestone briefings, and escalation paths) — apply throughout all phases.

Key roles are **Project Manager** (coordinates delivery, schedule, and communications), **Product Manager** (owns vision, backlog, and success metrics), **Developers** (implement and test features), **QA/Testing** (validate quality), and **Stakeholders** (provide inputs and approvals).
